tPigLoad

This component will be available in the Palette of the studio on the condition that you have subscribed to the relevant edition of Talend Enterprise Data Integration Big Data edition.

tPigLoad Properties

Component family

Processing/Fields

 

Function

This component allows you to set up a connection to the data source for a current transaction.

Purpose

The tPigLoad component loads original input data to an output stream in just one single transaction, once the data has been validated.

Basic settings

Schema and Edit Schema

A schema is a row description, i.e., it defines the number of fields that will be processed and passed on to the next component. The schema is either built-in or remote in the Repository. In this component, the schema is read-only.

 

 

Built-in: The schema will be created and stored locally for this component only. Related topic: see Talend Enterprise Studio User Guide.

 

 

Repository: The schema already exists and is stored in the Repository, hence can be reused in various projects and Job designs. Related topic: see Talend Enterprise Studio User Guide.

 

Mode

Select a mode from the list to run Pig scripts in Local mode and Mapreduce mode:

Local: All files are installed and run from your local host and file system.

Mapreduce - Apache: You need to precise the access to the Apache Hadoop to be used in this mode.

Mapreduce - Cloudera: You need to precise the access to the Cloudera Hadoop (CDH) to be used.

When selecting either of the two last options, you need to complete the following fields:

  • NameNode URI: type in the URI of the NameNode corresponding to the Apache or the Cloudera Hadoop to be used.

  • JobTracker host: type in the URI of the JobTracker corresponding to the Apache or the Cloudera Hadoop to be used.

For further information about HDFS, see http://hadoop.apache.org/hdfs.

 

Input file URI

Fill in this field with the full path to the input file.

 

Load function

Select a function from the list to determine how input data goes into Pig and comes out of Pig.

PigStorage: Loads and stores data in UTF-8 format.

BinStorage: Loads and stores data in machine-readable format.

TextLoader: Loads unstructured data in UTF-8 format.

  Field separator Type in the separator you need to use for the input data
Advanced settings Hadoop Properties Complete this table using the customized Hadoop parameters, if any, to override the default ones.
  Properties Complete this table using the customized Pig set configuration, if any, to override the default one. For example, the default_parallel key used in Pig could be set as 20.

tStatCatcher Statistics

Select this check box to gather the Job processing metadata at the Job level as well as at each component level.

Usage

This component is commonly used as intermediate step together with input component and output component.

Limitation

Knowledge of Pig script is required.
