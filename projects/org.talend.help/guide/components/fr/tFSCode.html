tFSCode

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition de Talend Enterprise MDM Studio. Pour plus d’informations concernant le mode Massively Parallel eXtension, consultez le Guide utilisateur de Talend Enterprise MDM Studio.

Propriétés du tFSCode

Famille de composant

FileScale

 

Fonction

Le tFSCode est un composant générique qui supporte n’importe quel code personnalisé FSLang. Ce composant traite des fichiers FileScale volumineux en temps réel. Afin d’optimiser les performances, le composant trie généralement les données avant de les traiter.

Objectif

Le tFSCode permet de créer des composants FileScale pouvant s’adapter à vos besoins spécifiques ainsi que d’exécuter des tâches de domaines spécifiques.

Basic settings

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma.

Si vous modifiez le schéma, il se met automatiquement en built-in.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et job designs. Voir également le Guide Utilisateur de Talend Enterprise Studio.

 

 

Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également leGuide utilisateur de Talend Enterprise Studio.

 

Input File Name

Nom et chemin d’accès du fichier contenant les données que vous voulez traiter.

 

Output File Name

Nom et chemin d’accès du fichier dans lequel vous souhaitez écrire les données traitées.

 

FSLang code

Saisissez le code FSLang adapté à vos besoins spécifiques. Pour plus d’informations à propos de FSLang, consultez Annexe A, Langage de programmation FileScale.

Advanced settings

Generate FSLang File

Cochez cette case afin de générer le fichier FSLang correspondant à votre Job, et cliquez sur le bouton [...] à côté du champ FSLang File Name pour spécifier son nom et son chemin d’accès.

 

Assign FileScale Path

Cochez cette case puis cliquez sur le bouton [...] à côté du champ FileScale Path pour sélectionner le fichier exécutable du programme FileScale qui vous permettra d’exécuter le composant.

 

Sort results

Cochez cette case afin de trier les résultats.

 

Custom FileScale Parameter (separated by ,)

Saisissez les paramètres pour chaque opération spécifique que vous souhaitez ajouter à l’appel de l’exécutable FileScale.

 

Specify Number of Process Child

Cochez cette case et saisissez le nombre de processus enfants à utiliser afin de mener à bien l’agrégation.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau de chaque composant ainsi qu’au niveau du Job.

Utilisation

Ce composant traite des fichiers, ainsi il ne nécessite pas de flux de données en entrée ou en sortie. Il sert à vérifier les données contenues dans des fichiers volumineux.

Limitation

La limitation dépend des limites de la mémoire physique et de l’architecture du processeur. Par exemple, la taille totale des fichiers traités ne peut pas excéder la limite du système de fichiers pour supporter les fichiers très volumineux (maximum de la valeur 64 bits signée).

Scénario associé

Aucun scénario n’est disponible pour ce composant.
