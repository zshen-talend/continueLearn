tFSPartitionFile

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition de Talend Enterprise MDM Studio. Pour plus d’informations concernant le mode Massively Parallel eXtension, consultez le Guide utilisateur de Talend Enterprise MDM Studio.

Propriétés du tFSPartitionFile

Famille de composant

FileScale

 

Fonction

Le composant tFSPartitionFile vous permet de partitionner des données en masse d’un fichier d’entrée avec les méthodes de partitionnement Hash ou Round-Robin avant de les écrire dans un fichier de sortie. Cela permet de gérer de très grandes tables.

Ce composant possède des fonctionnalités de partitionnement de fichiers à gros volume en temps réel. Afin d’optimiser les performances, le composant trie généralement les données avant de les traiter.

Objectif

Permet de partitionner des données en masse avant de les écrire dans un fichier de sortie.

Basic settings

Schema et Edit Schema

Un schéma est une description de lignes, i.e., il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Cliquez sur Edit Schema pour modifier le schéma.

Si vous modifiez le schéma, il se met automatiquement en built-in.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisé dans divers projets et job designs. Voir également leGuide utilisateur de Talend Enterprise Studio.

 

 

Built-in : Le schéma est créé et conservé pour ce composant uniquement. Voir également leGuide utilisateur de Talend Enterprise Studio.

 

Property type

Peut être Built-in ou Repository.

 

 

Built-in : Propriétés utilisées ponctuellement.

 

 

Repository : Sélectionnez le fichier de propriétés du composant. Les champs suivants sont alors pré-remplis à l’aide des données collectées.

 

Input File Name

Nom du fichier contenant les données que vous souhaitez partitionner.

 

Output File Name

Nom du fichier dans lequel vous souhaitez écrire les données partitionnées.

Les fichiers de sortie générés se verront ajouter un postfixe avec un numéro auto-incrémenté qui dépend du nombre de partitions défini.

 

Record separator (char)

Caractère, chaîne de caractères ou expression régulière pour séparer les enregistrements (lignes).

 

Field separator (char)

Caractère, chaîne de caractères ou expression régulière pour séparer les champs dans un enregistrement.

 

Header

Nombre d’enregistrements à ignorer au début du fichier.

 

Footer

Nombre d’enregistrements à ignorer à la fin du fichier.

 

Number of partitions

Nombre de partitions dans le fichier.

 

Partition

Sélectionnez dans la liste la méthode de partition que vous souhaitez utiliser :

Round-robin : Les enregistrements sont partitionnés via la méthode Round-Robin, afin que chaque partition contienne plus ou moins le même nombre de lignes et que la répartition de charge soit effectuée. Puisqu’il n’y a pas de clé de partition, les lignes sont distribuées au hasard dans toutes les partitions.

Hash : Les enregistrements sont hachés en partitions basées sur la valeur d’une colonne clé ou de colonnes clé sélectionnée(s) dans le schéma du fichier.

Cochez la case Partition Key correspondant à la colonne (aux colonnes) que vous souhaitez utiliser comme base de partitionnement des données.

Advanced settings

Generate FSLang File

Cochez cette case pour générer le fichier FSLang correspondant à votre Job et cliquez sur le bouton [...] à côté du champ FSLang File Name pour spécifier son chemin d’accès et son nom.

 

Assign FileScale Path

Cochez cette case puis cliquez sur le bouton [...] à côté du champ FileScale Path afin de sélectionner le fichier exécutable du programme FileScale nécessaire à l’exécution du composant.

 

Specify Number of Process Child

Cochez cette case et saisissez le nombre de processus fils à utiliser pour effectuer l’agrégation.

 

Custom FileScale Parameter (separated by ,)

Saisissez les paramètres pour toute opération spécifique que vous souhaitez ajouter à l’appel de l’exécutable FileScale.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du Job ainsi qu’au niveau de chaque composant.

Utilisation

Ce composant gère des fichiers et ne nécessite donc pas de flux d’entrée ou de sortie. Il est utilisé pour effectuer une jointure sur des données to dans des fichiers à gros volume.

Limitation

La limitation dépend des limites imposées par la mémoire physique et par l’architecture du processeur. Par exemple, la longueur totale des fichiers traités ne peut excéder la limite système pour les fichiers à gros volume (valeur maximum de 64 bits signés).

Scénario : Partitionner des données en masse via la méthode Hash avant de les écrire dans un fichier de sortie

Vérifiez que vous avez dézippé et sauvegardé localement le fichier exécutable FileScale fourni par Talend . Vous devez définir le chemin d’accès à ce fichier exécutable dans la vue Advanced settings du composant tFSPartitionFile .

Ce scénario Java décrit un Job utilisant le composant tFSPartitionFile pour partitionner rapidement des données très volumineuses, selon la méthode Hash et deux des colonnes d’entrée comme clés de partition.

  • Déposez les composants suivants de la Palette dans l’espace de modélisation graphique : tRowGenerator, tFileOutputDelimited et tFSPartitionFile.

  • Reliez d’abord le tRowGenerator au tFileOutputDelimited à l’aide d’un lien Row > Main puis au tFSPartitionFile à l’aide d’un lien Trigger > OnSubjobOk.

Dans ce scénario, le composant tRowGenerator va générer des données selon le schéma que vous avez défini dans la vue Basic settings et les envoyer au fichier d’entrée. Si la génération des données s’effectue sans erreur, le composant tFSPartitionFile partitionnera les données en six sous-ensembles selon la méthode de partitionnement définie.

  • Cliquez sur le tRowGenerator afin d’afficher sa vue Basic settings et définir ses propriétés.

  • Cliquez sur le bouton [...] à côté du champ RowGenerator Editor pour ouvrir l’éditeur dans lequel vous pouvez définir votre schéma.

  • Dans la partie supérieure de l’éditeur, cliquez sur le bouton [+] pour ajouter les colonnes que vous souhaitez écrire dans le fichier d’entrée.

  • Définissez le schéma et configurez les paramètres des colonnes.

    Dans ce scénario, le fichier contient cinq colonnes : id, firstname, lastname, city et age.

Vérifiez que vous avez défini la longueur de vos colonnes. Sinon, un message d’erreur s’affichera à l’exécution du Job.

  • Si nécessaire, cliquez sur l’onglet Preview dans la partie inférieure de l’éditeur pour afficher la vue correspondante, puis cliquez sur le bouton View pour afficher une partie des données générées.

  • Cliquez sur OK afin de valider votre schéma et fermer l’éditeur du tRowGenerator.

  • Cliquez sur le tFileOutputDelimited pour afficher sa vue Basic settings et définir ses propriétés.

  • Paramétrez les propriétés du tFileOutputDelimited.

Dans ce scénario, définissez des variables de contexte pour le chemin d’accès au fichier d’entrée. Vous pouvez créer des variables de contexte de différentes manières. Pour plus d’informations concernant la création et l’utilisation de variables de contexte, consultez le Guide utilisateur de Talend Enterprise Studio. Dans cet exemple, définissez les variables de contexte directement dans la vue Component.

  • Placez votre souris sur le champ que vous souhaitez paramétrer, File Name dans cet exemple, puis appuyez sur F5.

    Une boîte de dialogue s’affiche.

  • Donnez un nom à cette nouvelle variable et sélectionnez son type dans la liste Type.

  • Dans le champ Default value, saisissez la valeur de contexte que vous souhaitez utiliser pour le chemin d’accès au fichier d’entrée.

  • Cliquez sur Finish afin de valider vos modifications et fermer la boîte de dialogue.

    La variable nouvellement créée s’affiche dans le champ File Name et dans la vue Contexts.

  • Dans la vue Basic settings du tFileOutputDelimited, cliquez sur le bouton Edit schema afin d’afficher le schéma que vous avez défini dans l’éditeur et le modifier si nécessaire.

  • Cliquez sur OK pour fermer la boîte de dialogue.

  • Cliquez sur le tFSPartitionFile pour ouvrir sa vue Basic settings et définir ses propriétés.

  • Dans les listes Schema et Property Type, sélectionnez Built-In.

  • Cliquez sur le bouton Edit Schema pour afficher une boîte de dialogue dans laquelle vous pouvez définir les colonnes de votre schéma. Le schéma doit correspondre au schéma du fichier d’entrée.

  • Cliquez sur OK afin de fermer la boîte de dialogue.

    Les colonnes du schéma défini s’affichent dans le tableau Partition configuration.

  • Définissez le nom des fichiers d’entrée et de sortie à l’aide de la variable de contexte définie précédemment, en appuyant sur les touches Ctrl + Espace et en sélectionnant la variable dans la liste.

  • Définissez les séparateurs d’enregistrements et de champs, dans les champs Record Separator et Field Separator respectivement, puis les en-têtes et les pieds de page dans les champs Header et Footer, si besoin.

  • Dans le champ Number of partitions, saisissez le nombre de sous-ensembles que vous souhaitez créer, six dans ce scénario.

  • Dans la liste Partition, sélectionnez la méthode de partition que vous souhaitez utiliser, Hash dans ce scénario.

  • Dans le tableau Partition configuration, cochez les cases correspondant aux colonnes d’entrée que vous souhaitez utiliser comme clés de partition, id et firstname dans ce scénario.

  • Cliquez sur l’onglet Advanced settings pour configurer les propriétés avancées.

  • Cochez la case Assign FileScale Path pour afficher le champ FileScale Path puis parcourez votre répertoire jusqu’au fichier exécutable livré par Talend.

  • Sauvegardez votre Job et appuyez sur F6 pour l’exécuter.

Une barre de progression apparaît dans l’espace de modélisation graphique sous le composant tFSPartitionFile, pour afficher l’avancement de l’opération. Cette barre de progression vous montre combien les données volumineuses sont traitées rapidement.

Lorsque le pourcentage de la barre de progression atteint les 100%, les données sont partitionnées en six sous-ensembles comme il a été défini dans les paramètres du composant et écrites dans les fichiers de sortie.

Chaque ficher de sortie généré se voit ajouter un postfixe avec un numéro auto-incrémenté qui dépend du nombre de partitions que vous avez défini, six dans ce scénario.
